# 问题

自动驾驶的车为什么还不能像人一样开？不是因为传感器不够多，而是因为车没有"想象力"。

人开车的时候，脑子里一直在跑模拟：前面那辆车会不会突然变道？如果我现在加速，三秒后会在哪？下雨天刹车距离要多留多少？这些"脑内预演"让人能提前反应，而不是等事情发生了再处理。

自动驾驶的传统做法是把感知、预测、规划拆成流水线——先看清楚周围有什么，再猜别人会怎么动，最后算自己该怎么走。问题是：每个模块各管各的，信息传一次丢一次，到规划那一步已经丢了太多上下文。更要命的是，这套流水线不会"想象"——它不能在脑子里试跑一遍"如果我这样做会怎样"。

World model（世界模型）就是给车装上想象力。它把多传感器数据压缩成一个紧凑的内部状态，然后在这个状态上"向前滚"——模拟未来几秒的世界会变成什么样。车可以在脑子里试跑多条路线，挑最安全的那条再真正执行。

这篇综述梳理了 2019-2025 年间 250+ 篇论文，把自动驾驶世界模型的全貌画了出来：谁在做什么、做到什么程度、还差什么。

# 翻译

## 锚点：车的白日梦

想象你闭上眼睛，脑子里"播放"一段你熟悉的路——红绿灯在哪、哪里容易堵、行人喜欢从哪冒出来。你甚至能在脑子里"试驾"：如果我现在右转，会不会和那辆公交车抢道？

世界模型就是给自动驾驶车装上这种"闭眼想象"的能力。它不是在真实世界里试错（太危险），而是在一个压缩的内部世界里反复排练，然后把最好的方案拿出来执行。

整篇综述围绕三个问题展开：

1. 想象什么？——生成未来物理世界（把当前场景往前"滚"几秒）
2. 怎么决策？——行为规划（在想象出的未来里挑最好的路）
3. 想象和决策怎么配合？——预测与规划的交互（让两者形成闭环）

## 想象什么：四种"做梦"的方式

车要想象未来，首先得选一种"画面格式"。论文把生成未来场景的方法分成四条轨道：

```
+------------------+     +------------------+
|  Image-based     |     |  BEV-based       |
|  生成未来照片     |     |  生成俯视图       |
|  (像素级真实)     |     |  (规划友好)       |
+------------------+     +------------------+

+------------------+     +------------------+
|  OG-based        |     |  PC-based        |
|  生成3D占据格    |     |  生成点云         |
|  (体素级精细)     |     |  (几何保真)       |
+------------------+     +------------------+
```

**Image-based**：直接生成未来的摄像头画面。Dreamer 系列从游戏起步，DriveDreamer 把它搬到真实街道。扩散模型（Diffusion）是主力——给它当前画面+动作指令，它"画"出几秒后的街景。好处是逼真，坏处是像素不等于几何，规划器很难直接用。

**BEV-based**：把多个摄像头的画面"拍扁"成一张俯视图。FIERY 是开山之作，后来 GenAD 能根据轨迹指令生成未来 BEV。好处是规划器直接能用（车道线、障碍物一目了然），坏处是丢了高度信息。

**OG-based（占据格）**：把世界切成 3D 小方块，每个方块标记"有东西/没东西"。OccWorld 用 Transformer 预测未来的占据格变化。好处是几何精细到厘米级，坏处是计算量爆炸——密集体素吃内存。

**PC-based（点云）**：直接预测未来的 LiDAR 扫描结果。ViDAR 把图像和动作编码后自回归地解码未来点云。好处是保留了原始 3D 几何，坏处是点云稀疏且不规则，处理起来麻烦。

演化趋势很清晰：从重型确定性体素融合 → 轻量随机 token 化表示，从只做感知预测 → 加入记忆和规划条件，从单模态 → 多模态语言驱动。

## 怎么决策：两条路线

有了对未来的想象，车还得决定"我该怎么开"。规划方法分两大阵营：

**学习派**：用数据训练神经网络直接输出轨迹。RL 类（强化学习）让车在想象的世界里反复试错；LLM 类把驾驶当成"下一个 token 预测"——DrivingGPT 交替生成图像 token 和动作 token；Cost-volume 类先采样一堆候选轨迹，再用学到的代价图打分排序。

**规则派**：用物理公式和搜索算法保证安全。IDM（智能驾驶员模型）管纵向跟车；RRT/A* 管路径搜索；连续优化管平滑轨迹。好处是可审计、可证明安全，坏处是面对复杂城市交通时规则写不完。

现实中两派在融合：学习派用世界模型做"想象中的试错"来提升样本效率，规则派用学到的代价函数替代手写启发式。

## 想象和决策怎么配合：三代交互

这是论文最有洞察力的部分。它把"预测"和"规划"的耦合程度分成三代：

```
第一代：开环          第二代：不可控闭环      第三代：可控闭环
+--------+           +--------+             +--------+
| 数据 → 生成 |      | 感知 ←→ 规划 |       | 4D世界 ←→ 驾驶Agent |
| (回放录像)  |      | (自回归滚动) |       | (可编辑、可注入)    |
+--------+           +--------+             +--------+
不响应动作            响应动作但不可干预       完全可控的虚拟沙盒
```

**开环**：DriveGAN、DriveDreamer 生成逼真视频，但只是"回放"——你的动作不影响未来。像看电影，不是玩游戏。

**不可控闭环**：自回归世界模型（GAIA-1、OccWorld 等）能根据 ego 动作生成未来，预测和规划统一在一个生成循环里。但用户不能注入罕见事件、不能改物理规则、不能调回放速度。像一个你能操控主角但不能修改关卡的游戏。

**可控闭环**：最新一代。CARLA、UniSim、DriveArena、LimSim++ 等平台把交通模拟器和生成模型嫁接——你可以编辑场景、注入危险事件、调整物理参数、获得闭环评估分数。像一个完全可 mod 的沙盒游戏。

## 训练：怎么喂饱这些模型

三条路：

1. **自监督学习**：用 2D 渲染监督 3D 占据格预测，或用未来点云做伪标签。省标注，但精度还追不上全监督。
2. **大规模预训练**：把百万帧图像-LiDAR 序列编码到统一 BEV/体素空间，一个 backbone 服务检测、跟踪、预测、规划多个下游任务。
3. **数据生成**：用训练好的世界模型反过来"造数据"——生成罕见场景（暴雨、施工区、行人鬼探头）喂给感知和规划模型。

## 效果怎么样

几个关键数字：

- **4D 占据格预测**（Occ3D-nuScenes）：I²-World-O 和 T³Former-O 达到 SOTA，纯相机方案 T³Former-F 也很有竞争力
- **运动规划**（nuScenes）：UniAD+DriveWorld 比 UniAD 单独用 L2 误差降 33%、碰撞率降 39%；3D 占据格输入下 T³Former-O 碰撞率减半
- **点云预测**（OpenScene）：DFIT-OccWorld 把 Chamfer Distance 从 ViDAR 的 1.58 降到 0.70-0.76

# 核心概念

## 1. 世界模型（World Model）

一句话：一个能在脑子里"向前播放"世界的压缩模拟器。

类比：你玩过那种"脑内下棋"吗？不看棋盘，在脑子里走几步，预判对手会怎么应。世界模型就是车的"脑内棋盘"——把传感器数据压缩成一个紧凑状态，然后在这个状态上模拟"如果我这样做，世界会变成什么样"。

为什么重要：没有它，车只能"看到什么反应什么"（reactive）。有了它，车能"想到什么提前准备"（proactive）。这是从 L2 辅助驾驶到 L4 自动驾驶的认知跃迁。

## 2. 4D 占据格（4D Occupancy Grid）

一句话：把三维空间切成小方块、再加上时间轴，记录"每个方块在每个时刻是否被占据"。

类比：把世界想象成一个巨大的乐高积木盒。每个时刻拍一张"哪些格子里有积木"的快照，连续播放就是一部"积木世界的电影"。车的规划器可以直接查询："三秒后，我想去的那个格子会不会被别人占了？"

为什么重要：它是连接"想象"和"决策"的桥梁。图像太底层（像素不等于碰撞），BEV 太扁（丢了高度），只有 4D 占据格既保留了完整几何，又能直接回答规划器最关心的问题——"那里安不安全"。

## 3. 可控闭环模拟（Controllable Closed-loop Simulation）

一句话：一个你能随意编辑规则、注入事件、获得真实反馈的虚拟驾驶沙盒。

类比：从"看电影"（开环）到"玩固定剧情的游戏"（不可控闭环）到"玩创意工坊 mod 的沙盒游戏"（可控闭环）。你可以说"在这个路口加一个闯红灯的电动车"，世界模型会生成这个场景并让你的规划器应对。

为什么重要：真实世界的危险场景太稀少（你不能为了测试让真车去撞人），但安全验证偏偏需要大量危险场景。可控闭环让你在虚拟世界里"制造危险"来考验算法，是通往 L4 认证的必经之路。

# 洞见

世界模型的真正价值不是"预测准不准"，而是它把自动驾驶从"感知-决策流水线"变成了"想象-验证循环"——车不再是一个被动的反应机器，而是一个能在脑子里排练多种未来、主动选择最优行动的 agent。

这个转变的深层含义是：自动驾驶的瓶颈从"看得清"移到了"想得对"。传感器精度的边际收益在递减，但想象力的质量（时间一致性、物理合理性、可控性）还有巨大的提升空间。

# 博导审稿

选题没问题，时机也对。2024-2025 年世界模型在自动驾驶领域爆发，确实需要一篇系统的综述把局面理清楚。三层分类法（生成、规划、交互）比之前的综述更有结构感。

方法上，分类体系做得扎实。把生成方法按表示形式（Image/BEV/OG/PC）分，把规划按学习/规则分，把交互按闭环程度分——三个维度正交，读者容易定位自己的工作在哪里。表格做得很全，250+ 篇论文的方法、输入输出、数据集一目了然。

但有几个问题：

第一，综述的"综"有了，"述"偏弱。大量篇幅在罗列"谁做了什么"，缺少跨方法的深度对比和洞察。比如 OG-based 和 BEV-based 在什么场景下该选谁？论文没给出清晰的决策指南。

第二，性能对比部分（§6）只覆盖了几个任务，且不同方法的实验设置差异大（有的用 3D-Occ 输入，有的用纯相机），横向比较的公平性存疑。

第三，对"世界模型到底有没有用"这个根本问题回避了。UniAD+DriveWorld 比 UniAD 好 33%，但这到底是世界模型的功劳还是更大模型/更多数据的功劳？消融不够。

第四，写作上学术套话偏多，每个小节结尾的 "Summary" 和 bullet points 格式化严重，读起来像流水账。

判决：**weak accept**。分类体系有价值，覆盖面够广，但缺少让人眼前一亮的跨方法洞察，更像一本"电话簿"而非"思想地图"。

# 启发

**迁移**：论文里"可控闭环"的三代演化（开环→不可控闭环→可控闭环）是一个通用的系统成熟度模型。任何需要"在虚拟环境中验证策略"的领域（机器人操作、金融交易模拟、药物分子设计）都可以用这个框架审视自己处于哪一代，下一步该往哪走。

**混搭**：4D 占据格 + 扩散模型的组合思路可以迁移到室内机器人。把家庭环境建模为占据格，用扩散模型生成"如果机器人推了这个杯子，桌面会变成什么样"的未来状态，让机器人在脑子里排练操作序列再执行。

**反转**：论文暗示了一个反直觉的结论——更好的传感器不一定带来更好的驾驶，更好的"想象力"才是关键瓶颈。这意味着：与其花钱堆更多 LiDAR，不如投资训练更强的世界模型。对做系统架构决策的人来说，这是一个值得认真考虑的资源分配反转。
