# 问题

自动驾驶的大模型明明能看懂场景、能回答问题，为什么一到规划轨迹就拉胯？

症结在这里：现有方法把「理解」和「预测」割裂成两条路。一条路用文字推理——模型先把看到的东西翻译成语言描述，再从文字里推断该怎么开。另一条路用扩散模型生成未来画面——能想象前方会发生什么，但想象完了不知道怎么用。

文字推理的问题是：语言天生是抽象的。你用文字说「前方有车正在变道」，这句话丢掉了位置、速度、轨迹曲线这些规划必须的精确信息。从图像到文字再到轨迹，每次跨模态转换都在漏信息。

生成未来画面的问题是：扩散模型生成的图很好看，但它是个独立模块，跟规划器之间没有统一的表征。生成归生成，规划归规划，两者各说各话。

FSDrive 的思路是：让模型直接用「画面」来思考。不翻译成文字，而是先在脑子里「看到」未来会发生什么（生成未来帧），然后基于这个视觉想象来规划路径。一个模型同时当世界模型和规划器。

# 翻译

## 锚点：在脑子里先「看到」未来

想象你在开车。前方路口，你不是先用语言描述「有辆白车从左边来」，而是直接在脑子里「看到」两秒后那辆车会到哪里，然后本能地调整方向盘。FSDrive 做的就是这件事——让模型用画面思考，而不是用文字思考。

这个「视觉思维链」（Spatio-temporal CoT）是整篇论文的核心锚点：模型先生成未来几帧画面（带上车道线、3D检测框这些感知标注），相当于在脑子里模拟了一遍「接下来会发生什么」，然后基于这个模拟结果来规划轨迹。

## 怎么做到的：一个模型，两个身份

FSDrive 用一个 VLA（Vision-Language-Action）模型同时扮演两个角色：

```
                    +------------------+
  当前画面 -------->|                  |------> 未来画面（世界模型）
                    |   Qwen2-VL-2B   |
  历史画面 -------->|                  |------> 轨迹规划（逆动力学模型）
                    +------------------+
                           ^
                           |
                    视觉思维链（CoT）
                    = 未来帧 + 感知标注
```

具体分三步走：

**第一步：统一预训练，让模型既能生成又能理解。** 基于 Qwen2-VL-2B，用三个任务同时训练：生成未来帧（世界模型能力）、3D目标检测（空间理解）、车道线预测（道路结构理解）。这三个任务共享同一个视觉编码器和语言模型骨架，让模型在一个统一的表征空间里同时具备生成和感知能力。

**第二步：视觉思维链（Spatio-temporal CoT）。** 这是关键创新。推理时，模型不是直接从当前帧跳到轨迹输出，而是先生成未来 N 帧画面，并在每帧上标注 3D 检测框和车道线。这些生成的「带标注的未来画面」就是模型的思考过程——它在视觉空间里做推理，而不是在文字空间里。

**第三步：基于视觉推理结果规划轨迹。** 模型把自己生成的未来帧当作额外输入，结合当前观测，输出最终的轨迹点。相当于：先想象未来会怎样，再决定现在该怎么开。

## 效果怎么样

三个 benchmark 上都拿了最好成绩：

- nuScenes 轨迹规划：L2 误差 3s 时 1.43m（之前最好的 GenAD 是 1.70m），碰撞率从 0.31% 降到 0.10%
- NAVSIM（闭环评估）：PDMS 分数 88.1，超过之前的 SOTA（Hydra-MDP 的 87.7）
- DriveLM（语言理解）：总分 56.6，超过 Dolphins 的 42.0

关键消融实验显示：加上视觉 CoT 后，3s L2 误差从 1.68 降到 1.43，碰撞率从 0.19% 降到 0.10%。视觉思维链确实在帮模型做更好的决策。

# 核心概念

## 视觉思维链（Spatio-temporal Chain-of-Thought）

一句话：让模型用「生成未来画面」代替「写文字推理」来做中间思考步骤。

类比：你下棋时不会在脑子里写一段话「如果我走这步，对方可能走那步」，你是直接在脑子里「看到」棋盘变化后的样子。FSDrive 让模型也这样——先在视觉空间里「看到」未来几帧的样子（包括其他车在哪、车道线怎么走），再基于这个视觉想象做决策。

为什么重要：这是论文的核心贡献。没有它，模型就退化成普通的端到端规划器，性能明显下降（消融实验证实）。它解决了文字 CoT 的信息损失问题——视觉推理保留了空间精度。

## 统一预训练（Unified Pre-training）

一句话：用一个模型同时学会「生成未来」和「理解当下」，让两种能力共享同一套表征。

类比：一个人既会画画又会看图说话，而且画画和看图用的是同一个大脑区域。这比「一个人画画、另一个人看图、两人再商量」要高效得多。

为什么重要：如果生成和理解是两个独立模型，它们的表征空间不对齐，生成的未来帧对规划器来说就是「外语」。统一预训练让生成出来的画面和真实观测在同一个空间里，规划器能无缝使用。

# 洞见

视觉推理比语言推理更适合空间决策——因为空间问题的「思考过程」本身就是空间的。

文字 CoT 在数学、逻辑这些符号化任务上很强，因为思考过程本身就是符号化的。但自动驾驶的决策本质上是空间的——你需要判断的是「那辆车两秒后在哪」「我的车能不能从这个缝隙穿过去」。把这些空间推理硬翻译成文字，就像用文字描述一幅画再让人根据描述画出来——每次翻译都在丢信息。

FSDrive 的真正洞见不是「生成未来帧有用」（这个很多人做过），而是：**中间推理步骤的模态应该匹配最终决策的模态**。规划是空间任务，所以思考过程也应该在空间里完成。这个原则可以推广到其他领域——音乐生成的 CoT 应该是音频片段而非文字描述，蛋白质设计的 CoT 应该是结构片段而非序列描述。

# 博导审稿

选题眼光不错。把 CoT 从文字搬到视觉空间，这个方向是对的。自动驾驶的规划确实是空间问题，用空间表征做中间推理比用文字更自然。问题是真实的——VLM 做驾驶决策时的语义鸿沟确实存在，不是人造缺口。

方法上有巧劲。统一预训练让一个模型同时具备生成和理解能力，这比拼接两个独立模型优雅。视觉 CoT 的设计也合理——不是简单地生成未来帧，而是在生成的帧上叠加感知标注（3D框、车道线），让「想象」变得结构化、可用。

实验诚意中等偏上。三个 benchmark 都做了，消融实验也到位——拆掉 CoT、拆掉预训练各任务，都能看到性能变化。但有几个问题：

- baseline 对比里，跟纯视觉方法比（UniAD、VAD）很强，但跟同期的 VLA 方法比（比如最新的几个端到端大模型方案）覆盖不够全
- 2B 参数的模型在 nuScenes 上跑开环规划，这个设定的实际意义有限。真正的考验是闭环、是复杂场景。NAVSIM 的结果更有说服力，但只高了 0.4 分
- 生成未来帧的质量如何？论文没有单独评估生成质量（FID/FVD），如果生成的未来帧本身不准，基于它做规划岂不是在错误的想象上做决策？

写作功力中规中矩。结构清晰，但学术套话偏多。方法部分的公式堆砌感较重，核心直觉反而被淹没了。

**判决：weak accept。** 方向对、设计合理、结果不错，但缺少对生成质量的独立评估，闭环优势不够显著，2B 模型的实用性存疑。

# 启发

**迁移：模态匹配的 CoT 原则。** 如果你在做任何涉及空间/视觉/时序决策的 AI 系统，可以问自己：我的中间推理步骤是不是在用错误的模态？比如做视频编辑的 AI，与其用文字描述「把这个镜头的色调调暖」，不如直接生成一个色调调整后的预览帧作为中间步骤。这个「CoT 模态应该匹配任务模态」的原则是可以直接拿走用的。

**混搭：统一预训练 + 你的感知模块。** 如果你有一个已经训练好的感知模型（比如检测器或分割器），可以考虑把它的输出作为「结构化标注」叠加到生成的未来帧上，形成更丰富的视觉 CoT。FSDrive 只用了 3D 框和车道线，但如果你的场景需要语义分割、深度估计或光流，同样的框架可以扩展。

**反转：不要急着把所有东西都变成文字。** 当前 VLM/Agent 的主流范式是「万物皆文字」——把图像翻译成文字描述，用文字做推理，再把文字翻译回动作。FSDrive 提醒我们：这条路在空间任务上可能是弯路。如果你在做机器人控制、游戏 AI、或任何需要空间推理的系统，值得停下来想想：有没有办法让模型直接在原生模态里思考，而不是绕道文字？
