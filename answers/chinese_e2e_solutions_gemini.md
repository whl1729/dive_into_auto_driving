国内自动驾驶头部企业（华为、小鹏、理想、小米、地平线等）在端到端（End-to-End, E2E）技术路线上演进得非常迅速。目前的行业共识已经从最初的“分段式端到端”向“One Model（一段式 / 全局端到端）”以及“VLA（视觉-语言-动作）大模型”演进。

以下是各家企业的端到端方案架构、技术特点，以及对应的顶会论文成果与链接：

---

## 1. 地平线 (Horizon Robotics)

地平线是国内端到端学术界与工业界结合最紧密的弄潮儿之一，技术路径从“模块化端到端”演进到了“全局矢量端到端”与“多模态大模型”。

* **技术方案**：
* **UniAD 阶段**：提出以“规划为导向”的框架，把感知、追踪、预测、建图等多个网络用 Transformer 统一串联，信息无损传递。
* **VAD 阶段**：转向完全矢量化（Vectorized）的端到端建模，摒弃了稠密的网格或 BEV（鸟瞰图）特征空间，用矢量化的 Agent 轨迹和地图元素大大提升了计算效率和推理速度。
* **Senna 阶段**：探索全局端到端，将大视觉语言模型（VLM）与端到端驾驶结合，利用大模型的泛化能力处理长尾复杂场景。


* **代表论文与链接**：
* **UniAD (CVPR 2023 最佳论文)**: *Planning-oriented Autonomous Driving*
* [论文链接 (arXiv)](https://arxiv.org/abs/2212.10156)


* **VAD (ICCV 2023)**: *VAD: Vectorized Scene Representation for Efficient End-to-End Autonomous Driving*
* [论文链接 (arXiv)](https://arxiv.org/abs/2303.12077)


* **Senna (2024)**: *Senna: Bridging Large Vision-Language Models and End-to-End Autonomous Driving*
* [论文链接 (arXiv)](https://arxiv.org/abs/2412.02241)

## 2. 理想汽车 (Li Auto)

理想汽车在端到端量产落地上的速度极快，开创了国内 **“端到端 + 系统2（快慢思考）”** 的双系统范式。

* **技术方案**：
* **系统1（One Model 一段式端到端）**：输入摄像头图像和导航信息，直接输出行驶轨迹。模型内部不进行显式的目标检测和框选，实现了从传感器到轨迹的端到端特征无损流动。该方案参考了 **DriveVLM** 与 **UniAD** 的部分思想。
* **系统2（认知大模型）**：基于 VLM（视觉语言模型），用一个高参数量的多模态模型来充当人类的“慢思考”，负责识别复杂的交通场景（如施工路段、手势交警、特殊路标），并输出提示词（Prompts）来引导系统1，或直接参与逻辑决策。

## 3. 华为 (Huawei)

华为乾崑智驾在 **ADS 3.0** 中全面摒弃了传统的 BEV 分离式架构，迈入端到端时代。

* **技术方案**：
* 采用 **GOD（通用障碍物识别）网络** 与 **PDP（预测决策规控）网络** 的协同架构。
* 传统的方案是“感知 $\rightarrow$ 预测 $\rightarrow   决策 \rightarrow 规划”，华为 ADS 3.0 将预测、决策、规划和控制融合成一个统一的 PDP 网络。GOD 网络负责全维度的场景重构（不仅识别标准红绿灯、车辆，还能本能地识别未知异形障碍物），PDP 网络直接在此基础上生成端到端的物理世界轨迹。


* **技术关联**：华为在业界开源过诸如 **ST-P3**、**VAD** 相关的改进思路。由于华为的商业保密性，量产 ADS 3.0 的核心算法代码并未直接开源发表论文，但其架构本质上属于感知无损输入 + 预测规控一体化（Two-step End-to-End）的工业界变体。

## 4. 小鹏汽车 (Xpeng)

小鹏汽车是国内最早量产“端到端”方案的厂商之一，目前正朝着大模型规模法则（Scaling Law）以及 VLA（视觉-语言-动作）方向大力演进。

* **技术方案**：
* 小鹏引入了 **XBrain** 架构，主要由 **XNet（深度视觉感知网络）**、**XPlanner（神经网络规划大模型）** 组成。
* 在最新的“AI鹰眼纯视觉方案”及 VLA 探索中，小鹏致力于让模型不仅具备空间感知，更具备时空理解能力。由于 VLA 模型通常伴随着巨大的视觉 Token 计算量，小鹏联合北大提出了 **FastDriveVLA** 架构，通过即插即用的重构机制对视觉 Token 进行剪枝，在保证端到端规划性能（SOTA）的同时，大幅降低了计算延迟，解决了大模型上车的功耗与时延瓶颈。


* **代表论文与链接**：
* **FastDriveVLA (AAAI 2026)**: *FastDriveVLA: Efficient End-to-End Driving via Plug-and-Play Reconstruction-based Token Pruning*
* *(注：该论文由小鹏与北大联合于近期发表在顶会 AAAI 2026 上，详细展示了其大模型端到端的优化手段。)*

## 5. 小米汽车 (Xiaomi)

小米汽车虽然起步相对较晚，但凭借后发优势直接锁定了最前沿的端到端路线。

* **技术方案**：
* 小米主打 **“车端两段式端到端（底座模型）”**。第一段是感知端到端（将多相机输入转换为时空物理表征），第二段是规控端到端。
* 小米的一大技术特色是 **“时空大模型”（Space-Time Prediction Model）**。在传统的端到端规划中，轨迹容易陷入局部最优解。小米通过该模型预测周围环境在未来数秒内的连续时空演特征，以此来进行概率轨迹采样，从而使端到端在城市复杂会车、加塞场景下的决策更像老司机。


* **技术关联**：其技术原型和设计思想高度致敬了学术界近两年关于 **OccFormer**（占用网络 Transformer）和 **VAD** 的组合变形，强调用流场（Flow）和占用格（Occupancy）的时空连续性来规避端到端的“黑盒不可控性”。

## 总结：行业技术演进特征

| 企业 | 核心量产架构特征 | 学术理论/论文源头 |
| --- | --- | --- |
| **地平线** | 模块化端到端 $\rightarrow$ 矢量化端到端 $\rightarrow$ VLM | UniAD, VAD, Senna |
| **理想** | 系统1 (One Model) + 系统2 (VLM 慢思考) | DriveVLM |
| **华为** | GOD (通用感知) + PDP (预测决策规控一体化模型) | 工业级魔改两段式 E2E |
| **小鹏** | XNet + XPlanner $\rightarrow$ 极致剪枝的高效 VLA 大模型 | FastDriveVLA |
| **小米** | 感知生成 $\rightarrow$ 时空预测大模型 $\rightarrow$ 轨迹规划 | OccFormer / 轨迹概率采样 |

这些企业虽然对外营销的“名词”各有不同，但底层的演进逻辑是非常一致的：**逐步取消中间人工定义的规则和硬编码数据结构（如去掉 3D Bounding Box），扩大 Transformer 模型的端到端全通路，并引入多模态大模型（VLM/VLA）来解决长尾场景（Corner Cases）的理解问题。**
